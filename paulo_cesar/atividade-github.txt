Questão 03 – existe um conceito muito importante no github chamado de branch(ramoem engles).defina branch no âmbito do gith.Porque você utilizaria o branch? Para qual branch você enviou seu arquivo? E se quisesse enviar para novo branch, o que você faria diferente?

resposta – Branch nada mais e do que um braço do projeto em enghes que segnifica “ramo” e importante destacar que no nome branch sempre deve ter seu login de rede mais o numero da atividade,deixando claro o que ela faz . O git não armazena dados como uma série de mudanças ou deltas,mais sim como uma série de snapshots.quando se faz um commit no get, o git grarda um objeto commit que contém um ponteiro para o snapshot do conteudo que vovê colocouna área de seleção, o aoutor eos metadados da mensagem,zero ou mais ponteiros para o commit ou commits que são pais desde commit: nemhum pai para o commit inicial,um pai para um commit normale multiplos pais para commit que resultem de um merge de dois ou mais branches.
